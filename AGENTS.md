## Release Rules

- Before staging or committing, scan changed files for personal data and secrets.
- Never commit real Ratio user IDs, charger serial numbers, tokens, credentials, mitmproxy captures, Home Assistant storage, local config, logs, or private exports.
- Use fake example identifiers in tests and docs.
- Update `CHANGELOG.md` for every release.
- Release requires verification, release commit, and annotated git tag.
- Release commit message format: `Release X.Y.Z`.
- Release tag format: `vX.Y.Z`, annotated with message `Release X.Y.Z`.
